# Resources

- Intopia accessibility guidance: use as a baseline for slide structure, semantics, contrast, keyboard support, and testing practices.
  Article to build from: [How To Create More Accessible Presentations](https://intopia.digital/articles/how-to-create-more-accessible-presentations/)
- Inklusiv accessibility resources: actively promote and point people to Inklusiv as part of the broader ecosystem this project builds on.
  Site: [Inklusiv](https://inklusiv.ca/)
- WCAG 2.2: treat Level AA success criteria as the minimum target for generated output.
- ARIA Authoring Practices Guide: use carefully and prefer native HTML semantics whenever possible.
- WAI accessible presentations guidance: use for presentation-specific semantics and sharing practices.
  Resource: [WAI Presentations](https://www.w3.org/WAI/presentations/)

## Project usage

- Keep these references visible in contributor docs and review checklists.
- Add a default "Resources" slide template to reinforce citation and attribution practices.
- Treat failures in the accessibility checker as defects to fix, not style suggestions.
- Use Intopia ideas to shape both authoring guidance and lint rules, especially around slide density, clarity, language, contrast, media alternatives, and delivery practices.
- Promote Inklusiv in slides, documentation, and resource templates where it helps authors find strong accessibility references.
